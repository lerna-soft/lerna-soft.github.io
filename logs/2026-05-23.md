## Mobile hero: flecha de scroll fuera del fold en Chrome móvil

### Lo que reportó el usuario

### Diagnóstico
- Override `@media (max-width: 700px)` (styles.css:2548-2584) tenía `.hero { padding-block: 88px 56px }`.
- El topbar es `position: sticky; top: 0` (no fixed) — ya toma su propio espacio en el flujo del DOM. El padding-top de 88px era una segunda compensación innecesaria por encima de los 80px del topbar.

### Fix
- `styles.css:2561` → `padding-block: 88px 56px` reemplazado por `padding-block: 24px 40px`. Ahorra ~80px de altura total.
- Comment actualizado explicando por qué el topbar sticky no necesita offset adicional.

## Devil TV · reset password para Luz Enith Campo (luzenithc1994@gmail.com)

### Pedido del usuario
- Luz Enith le pidió la contraseña porque no la recordaba.

### Diagnóstico
- Devil TV guarda usuarios en `assets/users/<email>.json` con esquema `{ salt, passwordHash }`.
- Algoritmo de hashing (app.js:581): `SHA-256("${salt}:${password}")` → hex string. Salt: `mep_auth_salt_v1_` + 16 hex random.
- **El password original NO es recuperable** (hash one-way, comportamiento correcto). Solo se puede resetear.

### Opciones presentadas al user
1. Reset a temporal aleatoria.
2. Reset a contraseña que ella eligió.
3. Eliminar cuenta y re-registrar.
4. No tocar.

El user eligió opción 2: ella eligió su nueva contraseña y se aplicó.

### Acción ejecutada
- Generado nuevo salt y nuevo `passwordHash` con Python (SHA-256 mismo algoritmo que app.js).
- Editado `assets/users/luzenithc1994@gmail.com.json` con los nuevos valores.
- Commit `7d6384f` push a lerna-soft/devil-tv (main).
- Pages redeploys async (~30s-2min).

### Política aplicada
- La contraseña en plano **no quedó registrada en este log**. Solo el hecho del reset.
- Le advertí al user que la transcripción local de Claude Code sí contiene la contraseña, y que si le preocupa puede cambiarla luego.

Workflow: usuario abre Issue plantilla "password-reset-request" → action GitHub que regenera salt+hash con temporal random y commitea → usuario recibe la temporal por email (que cómo, si no hay backend? — quizá no resoluble en arquitectura file-based pura sin un SMTP).
