## Implementación · teléfono único + amistad bidireccional

Branches creadas (frontend y backend) con el mismo nombre:
**`feat/phone-unique-and-bidirectional-friends`**.

### Migraciones (frontend repo, ambas idempotentes)
- `migrations/013_users_phone_unique.sql` — UNIQUE INDEX parcial en
  `users.phone_e164 WHERE phone_e164 IS NOT NULL`. Permite múltiples
  cuentas sin teléfono pero rechaza duplicados con el mismo número.
- `migrations/013_friends_linked_user.sql` — agrega `linked_user_id UUID
  REFERENCES users(id) ON DELETE SET NULL` a `friends`, índice, y backfill
  por match email/phone contra users (excluyendo self-link).

### Backend (commit `ad84b15`)
- `users-repository.js#createUser` y `#updateUserProfile` envuelven el
  INSERT/UPDATE y mapean los 23505 según `error.constraint` a errores
  nombrados (`email_already_registered` / `phone_already_registered`).
- `routes/auth.js#register` y `#updateProfile` devuelven 409 con mensaje
  en español "Ese número de teléfono ya está registrado en otra cuenta".
- `friends-repository.js#linkFriendsToUser(newUserId)` — idempotente:
  (1) UPDATE friends.linked_user_id = newUserId para filas de OTROS users
      cuyo email/phone matchea el del nuevo user.
  (2) INSERT recíproco: por cada owner que tenía al nuevo user como
      contacto, crear una fila en la agenda del nuevo user con el owner
      como amigo. NOT EXISTS para no duplicar.
- Llamada desde `register` (después de createUser) y desde `updateProfile`
  (tras un cambio de teléfono). Falla silencioso (log) si algo sale mal,
  no bloquea el flujo principal.
- `mapRow` expone `linkedUserId` al cliente.
