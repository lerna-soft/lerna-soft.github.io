### Decisiones de diseño durante la sesión

- **`renderer.js` separado de `app.js`** porque `print.html` también lo necesita. Cargado en ambos como `<script src="renderer.js">`.
- **viewBox `0 0 400 200`** común para todos los templates excepto cuando el export lo override. Aspect ratio 2:1.
- **Palette roles**: `colors[0]` = texto principal, `colors[1]` = acento, `colors[2]` = surface secundario, `colors[3]` = fondo claro. Documentado en README contributing.
- **PNG con fonts**: depende de que el browser ya tenga las Google Fonts cacheadas. Si fallan, queda un fallback genérico. SVG es el path "perfecto" si el user quiere fidelidad.
- **Favicon square**: render con viewBox 400×400 cuando `opts.square=true`. Pendiente: el renderer todavía dibuja sobre 400×200 incluso si pasamos square; los templates centran lo suficiente como para que funcione razonablemente a 64px pero no es ideal. Si el user quiere refinarlo, hay que añadir variantes "square" a cada template.
- **Brand sheet hash**: base64 de JSON `{n, t, tpl, pal, f}` (solo IDs). Resolución hace fetch de `data/*.json` para los detalles. Mantiene la URL razonablemente corta.
